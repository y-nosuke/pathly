# 画面設計（UI/UX仕様）

Jetpack Compose による Android アプリの画面仕様。**マップを主役にした map-first 構成**を基本方針とする。

> 対応する要望: [requirements.md](../requirements.md) の「地図で確認したい」「リアルタイムに記録したい」「操作をシンプルにしたい」。

## 設計方針

- **map-first**: 記録画面・詳細画面はどちらも全画面マップを土台にし、操作や情報はその上に重ねる（オーバーレイ／ボトムシート）。マップを小さなカードに埋めない。
- **ワンタップ記録**: 記録の開始/停止は大きな丸い FAB 一つ。
- **将来の拡張を地図とシートに載せる**: 立ち寄り・写真・編集はマップ上のピンとボトムシートの行として足せる形にしておく。

## ナビゲーション構成

ボトムナビの2タブ（記録・履歴）。詳細画面は全画面オーバーレイで開き、その間はボトムナビを隠す。実装は Jetpack Navigation を使わず、`MainActivity` の単純な State 管理（`selectedTab` / `selectedTrack`）。

```text
┌─────────────────────────────┐
│                             │
│      各画面（全画面）         │
│                             │
├─────────────────────────────┤
│      [ 記録 ]  [ 履歴 ]      │ ← BottomNavigationBar（詳細表示中は非表示）
└─────────────────────────────┘
```

| タブ | 機能                  | アイコン         |
| ---- | --------------------- | ---------------- |
| 記録 | 全画面マップ＋GPS記録 | `ic_location_on` |
| 履歴 | 過去記録の一覧・統計  | `ic_list`        |

> 計画・設定タブは Phase 3 で追加予定（現状は枠のみの構想）。

## 画面設計詳細

### 1. 記録画面（ホーム / map-first）

全画面のライブマップ。自分の現在地（青ドット）を表示し、記録中は経路をリアルタイムに描画する。

```text
┌─────────────────────────────┐
│        ● 記録中 00:12:34     │ ← 記録中のみ：経過時間ピル（上部中央）
│                             │
│         [全画面 Google Map]  │ ← 現在地追従・青ドット・経路polyline
│         ～～～ 経路 ～～～      │
│                             │
│      ┌───────────────────┐   │
│      │ 1.8km      42      │   │ ← 記録中のみ：距離・地点数の統計カード
│      │ 移動距離   地点数    │   │
│      └───────────────────┘   │
│            ( ▶ / ■ )        │ ← 記録開始/停止 FAB（丸）
├─────────────────────────────┤
│      [ 記録 ]  [ 履歴 ]      │
└─────────────────────────────┘
```

- **FAB**: 停止中は ▶（`ic_play_arrow`・primary色）、記録中は ■（`ic_stop`・error色）。
- **記録中の表示**: 上部に経過時間ピル、下部に距離・地点数の統計カード。
- **権限がない場合**: マップ上に権限リクエストのカードをオーバーレイ表示（「位置情報を許可」ボタン）。
- **エラー**: 下部にエラーカードを表示。

対応実装: `presentation/tracking/TrackingScreen.kt`, `TrackingViewModel`（`currentTrack` をリアルタイム購読）。

### 2. 履歴画面

日付順の記録一覧。先頭に統計サマリー、記録中があればアクティブカードを表示。項目タップで詳細画面へ。

```text
┌─────────────────────────────┐
│ 外出履歴                      │
│ ┌─────────────────────────┐ │
│ │ 📊 記録数 / 総距離 / 総時間│ │ ← 統計サマリーカード
│ └─────────────────────────┘ │
│ ┌─────────────────────────┐ │
│ │ 🟢 記録中 …               │ │ ← 記録中があれば先頭に
│ └─────────────────────────┘ │
│ ┌─────────────────────────┐ │
│ │ 2026/07/19  1.8km …   🗑 │ │ ← 各記録（タップで詳細／削除）
│ └─────────────────────────┘ │
├─────────────────────────────┤
│      [ 記録 ]  [ 履歴 ]      │
└─────────────────────────────┘
```

対応実装: `presentation/history/HistoryScreen.kt`, `HistoryViewModel`。

### 3. 詳細画面（map-first）

全画面マップに軌跡を表示し、情報はドラッグ可能なボトムシートに畳む。

```text
┌─────────────────────────────┐
│ (←)                         │ ← 地図に重なる戻るボタン
│         [全画面 Google Map]  │
│         ～～～ 軌跡 ～～～      │ ← 開始(緑)・終了/現在地(赤/青)マーカー
│                             │
│ ┌─────────────────────────┐ │
│ │ ═══                     │ │ ← ドラッグハンドル
│ │ 2026年07月19日   ●記録中  │ │
│ │ 14:30 – 15:50 ・ 1時間20分 │ │
│ │ ┌────────┐ ┌────────┐   │ │
│ │ │ 3.2km  │ │  128   │   │ │ ← 移動距離 / 地点数
│ │ │移動距離 │ │ 地点数  │   │ │
│ │ └────────┘ └────────┘   │ │
│ └─────────────────────────┘ │ ← ボトムシート（引き上げで詳細）
└─────────────────────────────┘
```

- **マーカー**: 開始＝緑、終了＝赤、記録中の現在地＝青。軌跡は `TrackLineOrange` の polyline。
- **ボトムシート**: 日付・記録中バッジ・時刻範囲・所要時間・移動距離・地点数。マップのロゴ／カメラはシートに隠れないよう下部パディングで退避。
- **将来**: このシートに写真・立ち寄り・編集の枠を追加（Phase 2）。

対応実装: `presentation/history/TrackDetailScreen.kt`（`BottomSheetScaffold`）。

## デザイントークン

### カラー（暖色系・Material3）

`PathlyAndroidTheme` は Material3 ダイナミックカラー対応（Android 12+）。非対応端末では以下の暖色系スキームを使用。ダークモード対応済み。

| 役割           | 色                       |
| -------------- | ------------------------ |
| primary        | `PathlyOrange` (#FF6B35) |
| secondary      | `PathlyGreen` (#2ECC71)  |
| error          | `ErrorRed`               |
| 軌跡 polyline  | `TrackLineOrange`        |
| 開始マーカー   | 緑（HUE_GREEN）          |
| 終了マーカー   | 赤（HUE_RED）            |
| 現在地マーカー | 青（HUE_BLUE）           |

定義: `ui/theme/Color.kt`, `ui/theme/Theme.kt`。

### アイコン

Material Icons は不使用。`res/drawable` のベクター＋`painterResource` を使う（`ic_location_on` / `ic_play_arrow` / `ic_stop` / `ic_arrow_back` / `ic_list` / `ic_delete`）。
