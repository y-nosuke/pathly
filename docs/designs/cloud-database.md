# クラウドDB・同期設計（将来）

> **ステータス:** 将来の実装用（Phase 3）。現在は Android Room によるローカル完結。
> ローカルのデータモデルは [specs/model.md](../specs/model.md) を参照。

複数デバイス間の同期・カップル間共有・Web アクセスを担う、Supabase（PostgreSQL）側の設計方針。

---

## 全体方針

- **オフライン優先**: Android Room をメインデータストアとし、接続時に Supabase と同期する
- **同一データ構造**: Room と Supabase で同じ構造を維持する
- **UUID主キー**: 同期時の競合を避けるため主キーは UUID
- **タイムスタンプ更新管理**: 最終更新で競合解決

### 担当機能（Supabase 側）

- 複数デバイス間のデータ同期
- カップル間のリアルタイム共有
- Web アプリからのデータアクセス
- Row Level Security（RLS）によるアクセス制御

---

## 同期戦略

- **同期タイミング**: アプリ起動時 / ネットワーク復旧時 / 手動同期
- **競合解決**: タイムスタンプベースの最終更新優先（＋ユーザー手動選択オプション）
- **セキュリティ**: クラウドは暗号化保存、ローカルは Encrypted SharedPreferences、位置情報のアクセス制御
